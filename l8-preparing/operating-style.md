# The Senior Operating Style (keystone)

The one-page daily check. Everything else (roadmap, checklists, insights) hangs off this. Not a curriculum — a way to show up.

## One line

> Own real work, decide from the business WHY, finish it, prove with numbers.

## The 5 behaviors

```
1. OWN, don't wait
   See the path → act + document why. Stop asking permission for reversible stuff.

2. PLAN before you build
   15 min: goal, steps, risks, what I need from who + when, done-criteria. Then go.

3. MAP tech ↔ business
   Every task: why does this matter + what number proves it.

4. FINISH + measure
   Ship it fully. Record before/after. Half-done = no story.

5. LEARN by doing
   Real project teaches more than books. Study only fills the gap (system-design framework).
```

## Covers all three needs

```
ADAPT   → 1 + 2  (own + plan = trusted fast at a new job)
LEARN   → 5      (work IS the teacher)
SENIOR  → 3 + 4  (business framing + measured impact = the bar)
```

## No deadline — milestones, not pressure

No hard date. Senior arrives when evidence stacks (often >1yr — normal). Track progress, don't fail-by-calendar:

```
M1 ~mo2   Adapted: know codebase, shipping, trusted with real tasks
M2 ~mo4   Owning: ONE project I decide + drive (not assigned tasks)
M3 ~mo6   Delivered: that project shipped, metrics recorded
M4 ~mo9   Repeated: 2nd owned project, coordinated others
M5 ~mo12  Evidence stack: 2-3 owned wins w/ numbers + a deep-dive story
```

Miss a milestone → ask why, adjust. Not failure.

## Daily self-check

```
- Did I OWN/drive, or wait?
- Did I plan before building?
- Do I know the business WHY + the metric?
- Am I finishing, or leaving things half-done?
```
