# Planning Ahead — Practice Checklist

The #1 mid→senior gap: senior plans + drives; mid waits to be told. Run these until they're automatic. Not study — a daily work habit.

---

## Per-task mini-plan (run BEFORE coding — ~15 min)

- [ ] **Goal** — what outcome? (one sentence, in business terms if possible)
- [ ] **Steps** — break into 3-6 steps
- [ ] **Risks** — what could break? what's uncertain?
- [ ] **Dependencies** — what do I need from others, and BY WHEN?
- [ ] **Done =** — how do I know it's finished? (acceptance criteria)

Write it down. Then execute. (PR description, ticket, or journal — anywhere.)

---

## Move vs Ask (kill the "wait for alignment" habit)

Decide by reversibility:

- [ ] **Reversible + clear** → just DO it, document the why. (most things — stop asking permission)
- [ ] **Reversible + unclear** → pick one option, state your assumption, move
- [ ] **Irreversible / risky / costly** → propose a plan, get alignment FIRST (don't cowboy)

Rule: fast on reversible, careful on irreversible. Never treat everything as needs-permission.

---

## Drive, don't wait

- [ ] Path obvious? Act + document. Don't sit waiting for a blessing.
- [ ] Path unclear? Propose plan → "any objection?" → move if none.
- [ ] Blocked? Don't stall — find another path or escalate fast.
- [ ] See a risk coming? Flag it BEFORE it blocks you.

---

## Daily journal prompt (catch the pattern)

End of day, 2 min:
- [ ] Where did I DRIVE today? (decided + moved)
- [ ] Where did I WAIT when I could've moved?
- [ ] What did I see coming and handle early?

Track the drive-vs-wait ratio. Goal: more drive over time.

---

## The tell (am I doing it?)

```
Mid     "Finished the task. What next?"
Senior  "Saw X coming, planned for it, handled Y already,
         and I need Z from team-A by Friday or it blocks us."
```

Steering, not following = planning ahead.
