# Senior Engineer Roadmap — Work + Learning Combined

**Target:** reach strong Senior Software Engineer bar (~6 months).
**Rule #1:** Job comes first. Do the new job well — it trains most senior skills for free and becomes your interview evidence.
**Rule #2:** Learning ramps up only as you settle. No grind. Protect your life.

Bar + interview process: see `bar-and-process.md`.

---

## The core idea

Don't separate "work" and "study." The job IS the gym for the hardest senior skills (ownership, planning, stakeholder, real system design). Dedicated study only fills the gaps the job can't: structured system-design framework + mock practice.

```
WORK   → ownership, planning, stakeholder, impact, AI discipline, real project to deep-dive
LEARN  → system-design framework + mocks, interview-story polish
```

---

## 6-Month Plan

### Months 1-2 — ADAPT (job 100%, learning ~0)

**Work**
- Succeed in new role: learn codebase, domain, team. Deliver.
- Bake in senior habits from day one:
  - Plan ahead — write plan/risks before coding, don't wait for alignment
  - Own outcomes — drive tasks to done, decide + document why
  - Frame impact — WHY + measurable result, not just WHAT

**Learn**
- Almost nothing. Survive + ship.
- Only habit: **work journal**, 5 min/day → what you owned, decided, coordinated. (= future interview stories, near-zero effort)

**Target hrs/week:** ~0 study

---

### Months 3-4 — FOUNDATION (job + light study)

**Work**
- Settled now. Start owning a meaningful piece of a project (→ Eng Leadership deep-dive material).
- AI discipline in daily work (no extra time):
  - Hold plan mode to the end, never abandon
  - Reflect spec back when requirements change
  - Tests first/alongside AI-generated code
  - Security pass every time (parameterized queries, validate input)
  - Review AI output systematically — catch its bugs

**Learn (~3 hrs/week)**
- System design foundation: building blocks + tradeoffs
- Learn the fixed framework, memorize it:
  ```
  1. Clarify requirements (WHO/WHAT/SCALE/CONSTRAINTS)
  2. Scope down (top 3, state out-of-scope)
  3. High-level design (all major concerns named — TOP-DOWN)
  4. Deep dive (pick critical concern)
  5. Tradeoffs (justify every choice: why X not Y)
  6. Bottleneck / scale
  ```
- Use real work problems as design reps (free)

**Target hrs/week:** ~3 study

---

### Months 5-6 — INTERVIEW PREP (job + focused study)

**Work**
- Push owned project toward a clear, deep-dive-able outcome (the Eng Leadership story)

**Learn (~4-5 hrs/week)**
- Full system-design mocks, weekly, top-down, distributed-systems focus
- Eng-with-AI practice in the 45+45 format (build → inspect/debug/refine)
- Behavioral/values stories: pull from journal, polish to STAR, tag to values
- Eng Leadership: prep the deep-dive of your owned project

**Target hrs/week:** ~4-5 study

---

## What maps to which interview

```
System Design (60m)     ← months 4-6 framework + mocks
Engineering w/ AI (90m) ← daily work AI discipline + month 5-6 format practice
Eng Leadership (45m)    ← the project you own at work (months 3-6)
Behavioral (60m)        ← work journal → stories (all 6 months)
```

## What we DROP (no time in 6 months + full-time job)

```
DROP   open source, public blogging, broad domain mastery
DROP   exhaustive distributed-systems theory (learn only what mocks expose)
DROP   chasing Staff-level skills (org influence, multi-team) — not the target
```

## Priority if time slips

```
1. Do the job well        (non-negotiable — obligation + evidence)
2. Work journal           (cheap, high return)
3. System-design mocks    (the trainable gate)
4. Everything else        (drop without guilt)
```

---

## Weekly rhythm (months 3-6)

```
Daily    5 min journal (during work)
Daily    AI discipline on the job (no extra time)
Weekly   1 study block (~1hr reading OR 1 mock)
Monthly  self-review: am I planning ahead? owning? framing impact?
```

## Honest expectation

6 months + full-time job ≈ 3-4 hrs/week study realistically. Enough for system-design competence + clean stories IF focused. Not enough to also do extras — so don't. Job + system design + stories = clears the standard Senior bar.
